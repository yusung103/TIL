Spring & Spring Boot
====================

> 스프링은 Java 백엔드 개발에 있어 떼어놓을 수 없는 프레임워크이다.

## Spring
<u>엔터프라이즈용 Java 애플리케이션 개발을 편하게 할 수 있게 해주는</u> **오픈소스** 경량급 애플리케이션 프레임워크이다.

>  엔터프라이즈용 Java 애플리케이션 개발을 편하게 할 수 있게 해주는

기업은 웹 서비스를 운영하기 위해서 비즈니스 로직을 구현을 해야하는데 비즈니스 로직은 기업이 제공하는 서비스를 코드로 구현한 것으로, 사용자의요구사항을 해결하기 위한 실질적인 코드를 의미한다. **스프링이 등장하기 이전에는 비즈니스 로직을 구현하기 위해 기술 자체에 대한 공부를 추가적으로 해야했다.**  

스프링이 등장 이후 이전 기술에 비해 사용 방법이 상대적으로 덜 복잡해졌다. **따라서 개발 초기에 기본적인 설정과 적용시킬 기술들만 잘 선택을 한다면 기술보다는 애플리케이션의 로직 자체에 더 집중하여 비즈니스 로직을 구현가능 하다.**

<br>

> 오픈소스

스프링은 **모든 사용자에게 무료**로 열려있고, 필요하다면 스프링의 코드 일부를 수정하여 사용해도 무관하다. 오픈소스로 프로젝트를 공개해 놓으면 **여러 사람이 프로젝트의 코드를 사용해봄으로써 다양한 검증 과정을 거칠 수 있다**는 장점이 있다.  

하지만, 정해진 인원이 프로젝트의 개발과 관리를 맡는 것이 아니기 때문에, 프로젝트의 개발과 개선이 안정적이지 못하다는 단점 또한 있을 수 있다. 하지만 스프링은 오픈소스 프레임워크이지만, **개발과 개선이 보장**이 된다. 현재 스프링은 오픈소스로 배포되어 있지만 스프링소스(SpringSource)라는 IT기업에서 관리하고 있으며, **스프링의 소스 코드를 수정하거나 개선하는 일에는 스프링소스의 한정적인 인원만 참여 가능**하다.

<br>

## Spring Boot
스프링으로 애플리케이션을 만들 때에 필요한 설정을 간편하게 처리해주는 별도의 프레임워크이다.  


```spring
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
    <context-param>
        <param-anme>contextConfigLocation</param-name>
        <param-value/WEB-INF/spring>
        .
        .
        .
```

위 코드는 스프링으로 개발을 할 때에 설정해주어야 하는 설정 정보를 담고 있는 파일 중 일부인데, 매우 복잡해 보인다.

<br>

```springboot
spring.h2.console.enabled=ture
spring.h2.console.path=/console
spring.jap.generate-ddl=true
spring.jpa.show-sql=true
```
위 코드는 스프링 부트가 기존의 복잡한 설정을 간략화 한 코드이다.  
즉, 스프링 부트는 스프링 애플리케이션을 만들 때에 필요한 설정을 간편하게 처리해주는 별도의 프레임워크이다. 스프링 부트를 사용하면 기존에 어려운 초기 설정에 쏟아야 했을 시간과 노력을 절약하여 비즈니스 로직을 구현하는데에 집중할 수 있다.

<br>

## 요약
**Spring**은 설정 파일을 작성할 수 있고  
 
**Spring Boot**는 자동 설정을 제공하여 간편하게 개발할 수 있다.
